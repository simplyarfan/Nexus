## üìù Quick Reference - Status Values

Use these exact status values in Notion:

- **Backlog** - Not started, future feature
- **To do** - Ready to start working on
- **In Progress** - Currently being developed
- **On review** - Code review or testing phase
- **Done** - Completed and deployed

## üìù Quick Reference - Priority Values

Use these exact priority values in Notion:

- **Must have** - Critical features, core functionality
- **Should have** - Important features, high value
- **Could have** - Nice features, medium value
- **Nice to have** - Optional features, low priority
- **Will not have** - Explicitly out of scope
